A Deep Learning Framework for Image CaptioningThis project implements an advanced deep learning architecture for automatic image captioning, developed as part of an Advanced Machine Learning course. It leverages a custom attention mechanism to generate context-aware descriptive text for images.  📊 Dataset & Data ProcessingDataset: The project utilizes the MS-COCO dataset, specifically the train2014 images and annotations.  Data Sampling: To optimize training performance, the pipeline randomly samples 50% of the dataset, resulting in a working set of 41,391 images.  Text Cleaning: Captions are converted to lowercase, stripped of special characters, and bounded by startseq and endseq tokens.  Tokenization: A Keras Tokenizer is used to build a vocabulary, capped at the top 5,000 most frequent words to manage memory efficiently. Sequences are padded dynamically up to a maximum length of 35 words.  🧠 Model ArchitectureThe framework is built using TensorFlow/Keras and features a custom encoder-decoder architecture with spatial attention.  Feature Extraction (CNN Encoder): A pre-trained ResNet-50 model (with the top classification layer removed) extracts spatial feature grids from 224x224 input images. These features are passed through a Dense layer with a ReLU activation.  Bahdanau Attention: A custom attention layer computes alignment scores between the image features and the decoder's hidden state, generating a context vector that tells the model where to look for each word it generates.  RNN Decoder: The text generation branch uses an Embedding layer (dimension 256) and two stacked LSTM layers (512 units each). The LSTM layers process the concatenated context vector and the embedded previous word to predict the next word in the sequence.  ⚙️ Training ConfigurationCustom Training Loop: The model utilizes a high-speed, custom train_step function optimized with @tf.function.  Hyperparameters: The model is trained using the Adam optimizer with a learning rate of 0.001 and a Sparse Categorical Crossentropy loss function, heavily masked to ignore padding tokens.  Epochs & Batching: Configured to run for 10 epochs with a batch size of 64.  Crash-Proof Checkpointing: The training loop includes an emergency backup sequence that actively saves model weights and extracted features directly to a mounted Google Drive workspace, allowing training to resume seamlessly if interrupted.  📈 EvaluationInference: The notebook provides a custom evaluate_attention function that feeds the image features through the decoder step-by-step to generate captions until the endseq token is predicted.  Visual Output: Includes a visual testing block that plots the target image alongside the newly generated attention-based caption for qualitative review.
